# Jobsheet 8 -- MongoDB dan Database Server

## Kesimpulan

Melalui pelaksanaan Jobsheet 8 ini, saya memperoleh pemahaman sekaligus
pengalaman praktik dalam memanfaatkan MongoDB sebagai database NoSQL
pada aplikasi backend berbasis Node.js. Pada tahap awal, saya
mempelajari perbedaan utama antara database relasional (SQL) dan
database NoSQL, terutama terkait model penyimpanan data, fleksibilitas
skema, serta penggunaan struktur data berbasis dokumen.

Selama proses pengerjaan, saya berhasil menghubungkan aplikasi Node.js
dengan MongoDB dan memahami konsep dasar seperti database, collection,
dan document. Saya juga mempelajari penggunaan ObjectId sebagai
identitas unik untuk setiap data yang disimpan dalam MongoDB.

Selain itu, saya mempraktikkan secara langsung operasi CRUD (Create,
Read, Update, Delete) menggunakan MongoDB Native Driver. Operasi yang
dilakukan meliputi insertOne dan insertMany untuk menambahkan data, find
untuk membaca data, updateOne dan updateMany untuk memperbarui data,
serta deleteOne dan deleteMany untuk menghapus data sesuai dengan
kriteria tertentu.

Saya juga memahami bagaimana MongoDB bekerja secara asinkron dalam
menangani data, cara melakukan pencarian dan pembaruan data berdasarkan
kondisi tertentu, serta melakukan pengecekan hasil operasi database
menggunakan MongoDB Compass sebagai alat bantu visual.

Dengan menyelesaikan jobsheet ini, saya mendapatkan pemahaman yang
komprehensif mengenai peran database server dalam pengembangan aplikasi
backend, serta kemampuan untuk mengelola data secara efektif menggunakan
MongoDB sebagai solusi NoSQL yang fleksibel, efisien, dan scalable untuk
aplikasi web modern.
