## Deskripsi
Jobsheet ini membahas praktik penggunaan **Git** sebagai Version Control System serta proses **deployment aplikasi Node.js (Express.js)** ke platform cloud **Vercel**. Melalui kegiatan ini, mahasiswa mempelajari cara mengelola source code, menyimpan proyek pada GitHub, dan mempublikasikan aplikasi agar dapat diakses secara online.

## Ringkasan Materi
Dalam jobsheet ini dilakukan beberapa aktivitas utama:
- Inisialisasi repository Git dan pengelolaan commit
- Sinkronisasi repository lokal dengan GitHub
- Penggunaan file `.gitignore` untuk mengatur file yang tidak perlu dilacak
- Konfigurasi aplikasi Express.js agar kompatibel dengan environment produksi
- Deployment aplikasi ke Vercel melalui repository GitHub

## Tujuan
- Memahami konsep version control menggunakan Git  
- Mampu mengelola source code secara terstruktur dan aman  
- Mampu melakukan deployment aplikasi Node.js ke cloud hosting  
- Memastikan aplikasi dapat diakses melalui URL publik  

## Kesimpulan
Dari pelaksanaan jobsheet ini, dapat disimpulkan bahwa penggunaan Git dan GitHub sangat penting dalam pengembangan aplikasi modern karena memungkinkan pengelolaan perubahan kode secara rapi, kolaboratif, dan aman.  

Selain itu, proses deployment ke Vercel memberikan pengalaman nyata bagaimana aplikasi backend yang dikembangkan secara lokal dapat dipublikasikan ke internet. Dengan melakukan penyesuaian konfigurasi, aplikasi Express.js dapat berjalan stabil di lingkungan produksi.  

Secara keseluruhan, jobsheet ini memberikan pemahaman lengkap mengenai alur kerja pengembangan aplikasi backend mulai dari pengkodean, version control, hingga deployment.
